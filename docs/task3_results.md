# Задача 3 — Консолидация CSV из 3 рекламных сетей (сводный отчёт)

## 1. Скрипт: `Delta/task3_consolidate.py`

**Назначение**: Прочитать 3 CSV-выгрузки из разных рекламных сетей (Лист1, Лист2, Лист3 в исходной Google-таблице), обработать каждую согласно её уникальному формату, привести к единой схеме и вывести сводный отчёт с по-источниковой и кросс-источниковой сводкой.

**Этапы конвейера**:
1. Прочитать каждый CSV (UTF-8 BOM, `DictReader` с поиском столбцов по заголовку).
2. Обработать и нормализовать по-источниковые правила (формат даты, единицы измерения, валюта, названия кампаний).
3. Отсортировать все строки по дате, затем по названию кампании (канонический порядок: «РК 1», «РК 2», «РК 3»).
4. Вывести полную единую таблицу (27 строк деталей + строка ИТОГО).
5. Вывести по-источниковую сводку с агрегированными показами, кликами, затратами, CTR и CPM.
6. Вывести общий итог с задокументированными допущениями (курс конвертации USD → RUB для Лист3).

### 3.4 Нормализация названий кампаний

- **Целые**: Удалить пробелы и неразрывные пробелы (`\u00a0`) — разделители тысяч, затем `int()`.
- **Дробные/русская локаль**: Удалить символ `₽` и неразрывные пробелы, убрать пробелы-разделители тысяч, заменить десятичную запятую на точку, затем `float()`.

## 6. Примечания по форматированию Google Sheets

### 6.1 Рекомендуемая структура листа

Сводный отчёт должен быть организован так, чтобы его можно было **легко дополнять ежедневно** с минимальными изменениями формул:

**Вкладка: «Сводный отчёт» (основной лист)**

| Столбец | Содержимое | Рекомендуемая формула / подход |
|--------|---------|------------------------------|
| A | Источник | Исходные данные (можно заполнить через импорт) |
| B | Кампания | Исходные данные |
| C | Дата | Исходные данные (DD.MM.YY) |
| D | Показы | Исходные данные |
| E | Клики | Исходные данные |
| F | Затраты, руб | Исходные данные |
| G | CTR, % | `=E2/D2*100` (вычисляется) |
| H | CPM, руб | `=F2/D2*1000` (вычисляется) |

**Расположение данных**: Хранить все 27 строк в плоской таблице. Новые ежедневные данные добавлять ниже существующих строк. Использовать `FILTER()` или `QUERY()` для сводных агрегатов, чтобы новые строки автоматически включались.

### 6.2 Стратегия ежедневного дополнения

- **Подход с импортом**: Каждый исходный CSV можно импортировать в отдельную вкладку (Лист1, Лист2, Лист3), после чего сводная вкладка подтягивает данные через формулы `QUERY` или `{IMPORTRANGE}`.
- **Альтернатива**: При использовании Google Apps Script или ручной вставки сохранять тот же порядок столбцов в сводной вкладке. Новые ежедневные данные добавляются вниз.
- **BI-инструмент** (предпочтительно для zero-обслуживания): Подключить 3 исходных CSV к BI-инструменту (DataLens, Looker Studio) с автоматическим обновлением. Единая схема уже спроектирована; модель данных BI-инструмента обрабатывает агрегацию и ежедневное обновление автоматически.

### 6.3 Ключевые формулы для сводного раздела

- Итоги по источнику (при условии, что названия источников в столбце A, числа в D:F):
  - `=SUMIF(A:A, "Лист1", D:D)` — всего показов для Лист1
  - `=SUMIF(A:A, "Лист2", E:E)` — всего кликов для Лист2
  - и т.д.
- Общий итог:
  - `=SUM(D:D)` — всего показов
  - `=SUM(E:E)` — всего кликов
  - `=SUM(F:F)` — всех затрат
- CTR: `=total_clicks/total_impressions*100`
- CPM: `=total_cost/total_impressions*1000`

## 8. Экспорт CSV (готово для BI)

В `Delta/task3_consolidate.py` добавлена функция `export_csv()`, которая записывает нормализованные консолидированные данные в CSV-файл, готовый к импорту в Google Sheets, DataLens, Looker Studio или любой BI-инструмент.

### 8.1 Выходной файл

| Атрибут | Значение |
|-----------|-------|
| Путь к файлу | `Delta/consolidated_report.csv` |
| Кодировка | UTF-8 с BOM (`\ufeff`) — обеспечивает корректное отображение русских символов в Excel/Google Sheets |
| Разделитель | Запятая (`,`) |
| Формат чисел | Целые/дробные без разделителей тысяч |
| Десятичный разделитель | Точка (`.` — стандартная английская локаль) |

### 8.2 Схема

| Столбец | Тип | Пример |
|--------|------|---------|
| `Source` | `str` | `Лист1` |
| `Campaign` | `str` | `РК 1` |
| `Date` | `str` | `01.03.24` |

### 8.3 Структура файла

CSV содержит **28 строк** (1 строка заголовка + 27 строк данных):

```

Лист1,РК 1,01.03.24,57122,796,4776.0
```

Каждая из 27 строк деталей сводного отчёта (3 даты × 3 кампании × 3 источника) записывается как строка CSV. Порядок сортировки соответствует выводимой таблице: по возрастанию даты, затем по каноническому имени кампании (`РК 1`, `РК 2`, `РК 3`).

### 8.4 Как использовать

1. Запустить скрипт как обычно:
   ```
   python Delta/task3_consolidate.py
   ```
2. Файл `Delta/consolidated_report.csv` генерируется автоматически **до** вывода в консоль.
3. Файл **перезаписывается** при каждом запуске (не дополняется).
4. Импортировать CSV в ваш BI-инструмент:
   - **Google Sheets**: `Файл → Импорт → Загрузить → Заменить текущий лист`
   - **DataLens / Looker Studio**: Подключить CSV как источник данных и настроить ежедневное обновление.
5. Ручное форматирование не требуется — UTF-8 BOM обрабатывает русские символы, десятичные точки корректно парсятся во всех современных BI-инструментах.

### 8.5 Замечания по реализации

- Числа записываются **без разделителей тысяч**, чтобы BI-инструменты могли обработать их как числовые столбцы без настройки локали.
- Экспорт CSV **не влияет на остальные функции**: вывод в консоль (форматированная таблица и сводки) остаётся идентичным предыдущей версии.
- Функция использует только стандартную библиотеку Python (`csv`, `pathlib`) — без внешних зависимостей.
